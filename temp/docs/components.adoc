= Components
:toc: left

NOTE: Mimameithr simulates an arbitrary space body, not Earth.
No component assumes Earth-specific conditions.
Parameters drive all physical outcomes; Earth is one possible configuration, not the reference.

== Inter-Component Communication

One component owns each state field.
No other component may write to it.
Consumers read state as read-only.

This is a hard constraint.
Write ownership is not negotiated at runtime and not shared.

== Component Boundaries

Two axes determine whether a process is its own component or part of an existing one:

* **Data ownership** — does this process own distinct state that no other process may write?
If yes, it is a component.
* **Coupling frequency** — does this process tick at a different cadence than related processes?
If yes, it is a component.

Both axes must be considered together.
A process that shares state ownership with another is not a component regardless of cadence.
A process that owns distinct state but ticks at the same cadence as a related process may still warrant splitting if the state domains are unrelated.

_Example:_ Tectonics and Dynamics operate in the same physical domain but are separate components.
Tectonics owns the plate and fault map; Dynamics owns stress accumulation at defined points and publishes threshold events.
Neither writes the other's state.
Splitting them reflects both distinct ownership and different tick behaviour.

When in doubt, split.
Merging is harder to undo than splitting.

Components are grouped by the sphere they belong to.
The climate system is the interaction between five spheres (atmosphere, hydrosphere, cryosphere, lithosphere, biosphere); each interacts with the other four.
Primosphere sits outside that web — it is an external driver that the climate system receives from but does not feed back into.
Civilization (anthroposphere) is layered on top of the climate system once active.

== Primosphere

External driver.
Provides the physical and orbital context all other components depend on.
Not part of the climate system interaction web.

=== Albedo

=== Domain

Defines the intrinsic physical properties of the simulated body.

=== Irradiance

Quantifies the stellar energy flux received at each point of the domain's surface.
Driven by the orbital position exposed by <<Motion>> and the axial tilt exposed by <<Domain>>.

=== Motion

Tracks the orbital mechanics of the domain relative to its parent body.
Governs how distance and angular position evolve over time.
