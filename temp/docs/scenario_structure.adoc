= Scenario Structure
:toc: left

Defines the folder structure and naming rules for scenario files.
The parser uses this structure and naming to determine what to parse and against which schema.
This is a parsing contract, not a convention.

== Folder Structure

----
scenarios/
└── <scenario>/
    ├── scenario.toml
    └── <domain>/
        └── <sphere>/
            └── <component>.toml
----

Every level is required and load-bearing:

* `<scenario>` — designer-assigned scenario identifier.
* `scenario.toml` — scenario-level metadata (header, description, simulation window).
* `<domain>` — designer-assigned entity identifier.
The filename is the key; the parser derives the domain identity from it.
* `<sphere>` — sphere name, matches a known sphere identifier (e.g. `primosphere`, `lithosphere`).
* `<component>.toml` — component name, matches a known component under that sphere.
The dynacore is selected by the parser based on the component file contents.

Scenarios are self-contained.
No cross-scenario references exist; a domain identifier is only meaningful within its scenario.

== Templates

Templates define the available dynacores per component.
A dynacore is explicitly declared by the designer through the template used — there is no implicit selection.

=== Primosphere

==== Domain

[cols="1,3"]
|===
|Dynacore |Description

|`disc` |Flat circular disc with uniform downward gravity.
|`sphere` |Spherical body with surface gravity derived from mass and radius.
|===

==== Motion

[cols="1,3"]
|===
|Dynacore |Description

|`elliptical` |Standard Keplerian orbit around a parent body. Covers circular orbits at eccentricity 0.
|`fixed` |Stationary at explicit coordinates. No orbital relationship.
|`binary` |Two bodies in mutual orbit around a shared barycentre.
|`resonant` |Co-orbital or Trojan configuration; orbital resonance with a sibling body.
|`unbound` |No position, no orbital relationship. Exists outside any coordinate system.
|===

=== Lithosphere

==== Crust

[cols="1,3"]
|===
|Dynacore |Description

|`mobile-lid-sphere` |Tectonic plate simulation on a sphere; models plate motion, stress accumulation, and fault generation.
|===

==== Terrain

[cols="1,3"]
|===
|Dynacore |Description

|`handcrafted` |Designer-authored elevation field; no procedural generation.
|`noise` |Procedural elevation field from noise functions; no tectonic simulation.
|`tectonic-noise-sphere` |Tectonic-informed elevation on a sphere, refined with noise.
|`tectonic-sphere` |Tectonic-derived elevation on a sphere; no noise refinement.
|===
