= Mimameithr — Architecture
:toc: left
:toclevels: 3

== Execution model

The engine operates in two distinct phases with different execution paradigms.
Both phases call the same stateless Kvasir simulation functions.
The orchestration layer differs.

== Urth — Generation phase

`Urth` is the generation phase. The name is taken from Urðr, the Norn of the past — the world before sentient civilisation begins.

`Urth` lives in Nornir.

=== Urth engine

The `Urth` engine drives the generation phase.
It owns the dependency graph between components and is responsible for propagating changes.
It is fully operable without Godot — tests can drive it directly in C#.

Urth is driven by the player, not by time.
It uses an evaluation graph, not a tick loop.

When the player modifies a setting on a body:

. The engine invalidates all nodes downstream of that setting in the dependency graph.
. It re-evaluates invalidated nodes in topological order.
. Each node calls the relevant Kvasir function directly with the current settings as input.
. The result is written back to the component state.

There is no tick, no elapsed time, and no compression during this phase.
Modifications are applied and propagated immediately.

=== API contract

The engine exposes a CQRS interface.

*Commands* — one per system.
Each command takes an `EntityId` and the complete settings block for that system.
Settings are always replaced in full; partial updates are not supported.
A command triggers invalidation and re-evaluation of all downstream nodes.

Examples:
----
ConfigureTectonics(EntityId, TectonicsSettings)
ConfigureOrogeny(EntityId, OrogenicSettings)
----

*Queries* — return the current computed state of a component for a given `EntityId`.
Queries have no side effects.
Results are readable directly from C# without any presentation layer.

=== Static inputs

Orbit and BodyRotation are configuration values during generation, not active systems.
Their settings are present as inputs to downstream computations but are not evaluated as systems.
They become active only when the phase transition occurs.

=== Iterative systems

Some future systems (orogeny, erosion, climate) may require multiple passes to converge.
When that is the case, temporal compression will apply within the generation phase for those passes.
The compression range for iterative geological passes will differ from the range used during the active phase.
Exact ranges are deferred until the relevant systems are implemented.

== Phase transition

The player declares the planetary system ready.
This declaration is the simulation epoch — the reference point from which all subsequent time is measured.

At this point:

* The state produced by Urth becomes the initial ECS state.
* The Verdandi tick loop starts.
* Orbit and BodyRotation become active systems.
* All bodies in the system advance on a shared timeline.

== Verdandi — Active phase

=== Verdandi engine

The `Verdandi` engine drives the active phase.
It runs a standard ECS tick loop.
Each tick, systems read component settings and current state, call Kvasir, and write updated state back.
Time is the driver.

Temporal compression is available but constrained to ranges appropriate for sentient history — tens of thousands of years at most.

== System development sequence

Systems are ordered by timescale and dependency.
One-shot systems produce a fixed substrate evaluated once during generation.
Iterative systems require multiple passes to converge; the number of passes is left to the player.

=== Priority 1 — Substrate (one-shot, Myr+)

Tectonics → Orogeny → Elevation → Geology → Basin → Sedimentation → Geothermal → Dynamics

Evaluated once. Serve as fixed inputs for all subsequent systems.

=== Priority 2 — Energy

Irradiance → Albedo → Surface Temperature → Thermal Redistribution

=== Priority 3 — Atmosphere, water, and ice

Circulation → Precipitation → Sea Level → Ocean Circulation → River Network → Groundwater → Ice Sheet → Glacier → Sea Ice

These systems form a closed loop via the Albedo feedback: Flora and ice cover modify Albedo, which modifies Thermal Redistribution, which propagates through the entire chain.
Layers 2 and 3 must converge before the biosphere is stable.

=== Priority 4 — Biosphere

Biome → Flora → Soil → Fauna → Disease

=== Priority 5 — Mann

Conceptual. Deferred.
