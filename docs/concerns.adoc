= Concerns
:toc: left

NOTE: All physical quantities via UnitsNet — no raw `double` for science values.

NOTE: Elements are named after Old Norse terms. See `glossary.adoc` for the full list with original forms and descriptions.

NOTE: Boundary processes (energy or matter transfer between two elements) are assigned to the *receiver*, not the emitter.
Example: surface-to-air heat transfer belongs to Lopt, not Hlothyn.

NOTE: Element boundaries are organizational, not hard constraints.
Concerns may be split, merged, or reassigned as implementation reveals better boundaries.
The element a concern belongs to reflects current understanding, not a permanent contract.

== Geimr

Celestial mechanics at body scale.
Covers the astronomical properties of space bodies — their geometry, orbital dynamics, and rotation.
Concerns here operate on body entities, not cells.
Forms the root dependency layer for all other elements.

=== Irradiance

==== Owns

* Insolation per R0 cell — `Irradiance` (W/m²)
* Solar zenith angle per R0 cell — `Angle`
* Day/night state per R0 cell

==== Reads

* Radius ← Body geometry
* Axial tilt ← Body geometry
* Current rotation angle ← Rotation
* Current distance from star ← Orbit
* Current orbital angle ← Orbit
* Luminosity ← Stellar luminosity

==== Exposes

_To be filled when consumers are designed._

==== Notes

* Operates at R0 — 122 cells, full sphere coverage guaranteed by contract.
* Smooth latitudinal function; R0 is the correct resolution.

_To be filled when consumers are designed._

=== Tectonics

==== Owns

* Crust composition per cell — felsic / mafic (enum)
* Crustal thickness per cell — `Length`
* Vertical displacement rate per cell — `Speed` (m/yr)
* Boundary type per cell — convergent / divergent / transform / none (enum)

==== Reads

_Nothing, external driver._

==== Exposes

* Crust composition → Elevation
* Crustal thickness → Elevation
* Vertical displacement rate → Elevation
* Boundary type → Elevation, Dynamics, Geothermal

==== Notes

* One-shot computation at world gen. Only changes via explicit forcings.
* Timescale: Myr.

=== Orogeny

==== Owns

* Orogenic age per cell — `Duration`
* Accumulated crustal shortening per cell — `Length`
* Relief potential per cell — `Ratio` (0–1, decays with age)

==== Reads

* Boundary type ← Tectonics
* Vertical displacement rate ← Tectonics

==== Exposes

* Orogenic age → Elevation
* Accumulated crustal shortening → Elevation
* Relief potential → Elevation
* Orogenic age → Geology

==== Notes

* HYPOTHESIS: age + relief potential is sufficient to distinguish old vs young mountains. Validate when Elevation is implemented.
* One-shot computation at world gen. Only changes via explicit forcings.
* Timescale: Myr.

=== Elevation

==== Owns

* Crust composition ← Tectonics

* Boundary type ← Tectonics
* Orogenic age ← Orogeny
* Accumulated crustal shortening ← Orogeny
* Relief potential ← Orogeny

==== Exposes

_To be filled when consumers are designed._

==== Notes

* Derived from isostatic balance + tectonic displacement.
* One-shot computation at world gen. Only changes via explicit forcings.
* Timescale: Myr.

=== Dynamics

==== Owns

* Seismic frequency index per cell — `Ratio`
* Seismic intensity index per cell — `Ratio`
* Seismic duration index per cell — `Ratio`
* Volcanic frequency index per cell — `Ratio`
* Volcanic intensity index per cell — `Ratio`
* Volcanic duration index per cell — `Ratio`

==== Reads

* Boundary type ← Tectonics
* Vertical displacement rate ← Tectonics
* Crust composition ← Tectonics

==== Exposes

* Volcanic frequency index → Geothermal
* Volcanic intensity index → Geothermal

==== Notes

* Index values are designer-scaled (1–10), not physical quantities. Scaling driven by config.
* One-shot computation at world gen. Only changes via explicit forcings.
* Timescale: Myr.

== Mann

Sentient civilisation at cell and network scale.
Covers settlement, resource extraction, infrastructure, and cultural dynamics.
Emerges from and is constrained by all other elements.

== Projections

Read-only aggregations derived from one or more elements.
No simulation logic — pure views over existing concern state.
Do not belong to any element.

=== Surface cover

Aggregates per-cell surface composition from all elements that expose a surface state.
Single consumer: Albedo.
Fed by: Hrim (Ice Sheet, Sea Ice), Ithunn (Flora), Logr (Ocean), Hlothyn (terrain) — as those elements are designed.
