NOTE: All physical quantities via UnitsNet — no raw `double` for science values.

== Aethersphere

Celestial mechanics at body scale.
Covers the astronomical properties of space bodies — their geometry, orbital dynamics, and rotation.
Concerns here operate on body entities, not cells.
Forms the root dependency layer for all other spheres.

[mermaid]
....
flowchart TB
    subgraph Aethersphere
        StellarLuminosity([Stellar luminosity])
        BodyGeometry([Body geometry])
        Orbit([Orbit])
        Rotation([Rotation])
    end

    Irradiance([Irradiance])

    BodyGeometry --> Rotation
    StellarLuminosity -.-> Irradiance
    BodyGeometry -.-> Irradiance
    Rotation -.-> Irradiance
    Orbit -.-> Irradiance
....

=== Stellar luminosity

==== Owns

* Luminosity — `Luminosity`

==== Reads

_Nothing, external driver._

==== Exposes

_To be filled when consumers are designed._

=== Body geometry

==== Owns

* Radius — `Length`
* Axial tilt — `Angle`

==== Reads

_Nothing, external driver._

==== Exposes

_To be filled when consumers are designed._

==== Notes

* Static or near-static, set at world gen, rarely changes.

=== Orbit

==== Owns

* Current distance from star — `Length`
* Current orbital angle — `Angle`

==== Reads

_Nothing, external driver._

==== Exposes

_To be filled when consumers are designed._

=== Rotation

==== Owns

* Rotation rate — `RotationalSpeed`
* Current rotation angle — `Angle`

==== Reads

_Nothing, external driver._

==== Exposes

_To be filled when consumers are designed._

==== Notes

* Computed state, ticks every step.
* Rotation rate is stable; angle is derived from rate × elapsed time.

== Pyrosphere

Surface energy input at cell scale.
Covers how energy from the star arrives at and is absorbed by each surface cell.
Concerns here operate on R0 cell entities.
Forms the thermal foundation for Aerosphere and Hydrosphere.

[mermaid]
....
flowchart TB
    subgraph Pyrosphere
        Irradiance([Irradiance])
        Albedo([Albedo])
        SurfaceTemperature([Surface temperature])
    end

    StellarLuminosity([Stellar luminosity])
    BodyGeometry([Body geometry])
    Orbit([Orbit])
    Rotation([Rotation])

    StellarLuminosity -.-> Irradiance
    BodyGeometry -.-> Irradiance
    Orbit -.-> Irradiance
    Rotation -.-> Irradiance
    Irradiance --> SurfaceTemperature
    Albedo --> SurfaceTemperature
....

=== Irradiance

==== Owns

* Insolation per R0 cell — `Irradiance` (W/m²)
* Solar zenith angle per R0 cell — `Angle`
* Day/night state per R0 cell

==== Reads

* Radius ← Body geometry
* Axial tilt ← Body geometry
* Current rotation angle ← Rotation
* Current distance from star ← Orbit
* Current orbital angle ← Orbit
* Luminosity ← Stellar luminosity

==== Exposes

_To be filled when consumers are designed._

==== Notes

* Operates at R0 — 122 cells, full sphere coverage guaranteed by contract.
* Smooth latitudinal function; R0 is the correct resolution.

=== Albedo

==== Owns

* Albedo per R0 cell — `Ratio`

==== Reads

_Nothing, external driver for now. Will read surface composition when designed._

==== Exposes

_To be filled when consumers are designed._

=== Surface temperature

==== Owns

* Surface temperature per R0 cell — `Temperature`

==== Reads

* Insolation ← Irradiance
* Albedo ← Albedo

==== Exposes

_To be filled when consumers are designed._

== Anthroposphere

Human civilization at cell and network scale.
Covers settlement, resource extraction, infrastructure, and cultural dynamics.
Emerges from and is constrained by all other spheres.

== Projections

Read-only aggregations derived from one or more spheres.
No simulation logic — pure views over existing concern state.
Do not belong to any sphere.

=== Surface cover

Aggregates per-cell surface composition from all spheres that expose a surface state.
Single consumer: Albedo.
Fed by: Cryosphere (Ice Sheet, Sea Ice), Biosphere (Flora), Hydrosphere (Ocean), Gaiasphere (terrain) — as those spheres are designed.
