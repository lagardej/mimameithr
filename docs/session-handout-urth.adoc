== What was decided

=== Urth is the generation phase engine

* Lives in `Nornir/Urth/`
* Driven by the player, not by time
* Uses an evaluation graph, not a tick loop
* Fully operable without Godot — tests can drive it directly in C#

=== Execution model

When the player modifies a setting on a body:

. The engine invalidates all nodes downstream of that setting in the dependency graph.
. It re-evaluates invalidated nodes in topological order.
. Each node calls the relevant Kvasir function directly with the current settings as input.
. The result is written back to the component state.

=== API contract (CQRS)

*Commands* — one per system, replace settings in full:
----
ConfigureTectonics(EntityId, TectonicsSettings)
ConfigureOrogeny(EntityId, OrogenicSettings)
----

*Queries* — return current computed state for a given `EntityId`, no side effects.

=== Static inputs during generation

Orbit and BodyRotation are configuration values, not active systems.
They are present as inputs to downstream computations but not evaluated.

=== One-shot systems (Priority 1 — substrate)

These systems are evaluated once and serve as fixed inputs for all subsequent systems:

Tectonics → Orogeny → Elevation → Geology → Basin → Sedimentation → Geothermal → Dynamics

Currently implemented in Kvasir: *Tectonics* (two-pass, one-shot, confirmed stateless).
Orogeny, Elevation, Geology are the logical next steps.

=== Phase transition

When the player declares the system ready:

* Urth state becomes the initial ECS state for Verdandi
* The epoch is set
* Orbit and BodyRotation become active systems

== What exists already

* `Nornir/Urth/` — directory exists, empty
* `Kvasir` — TectonicsSimulation implemented and confirmed stateless pure function
* Three Nornir systems already exist: BodyRotation, Orbit, Tectonics
* ECS architecture and component conventions (`[Setting]`, `[State]`) established

== Where to start

. Implement the CQRS command interface for the existing systems (starting with Tectonics)
. Wire Tectonics: command → invalidate → call `TectonicsSimulation.Run()` → write state
. Add a query for Tectonics state
. Cover with headless C# tests — no Godot required
